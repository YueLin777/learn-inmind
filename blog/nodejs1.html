    # nodejs 是什么？

    • Node.js是一个开源的，跨平台的JavaScript运行环境
    • 通俗来讲：Node.js就是一款应用程序，是一款软件，它可以运行JavaScript

    • Node.js 使用了一个事件驱动、非阻塞式 I/O 的模型，使其轻量又高效
    • Node.js 的包管理器 npm，是全球最大的开源库生态系统
    • Node.js 是一门动态语言，运行在服务端的 Javascript

    # Buffer

    # 简介

    Buffer 是一个类似于数组的 对象 ，用于表示固定长度的字节序列

    Buffer本质是一段内存空间，专门用来处理 二进制数据 .

    # 特点

    1. Buffer 大小固定且无法调整
    2. Buffer 性能较好，可以直接对计算机内存进行操作
    3. 每个元素的大小为 1 字节（byte）= 8 bit

    # 使用

    # 创建

    在node.js中创建 Buffer 的方式

    方式一

    //TODO 方式一：

//创建 一个长度 为 10 字节的Buffer，相当于申请了 10字节的内存空间，每个字节的值为0
//<Buffer 00 00 00 00 00 00 00 00 00 00>
let buf_1 = Buffer.alloc(10)
console.log(buf_1)

    方式二：

    //TODO 方式二：
//创建 一个长度 为 10 字节的Buffer
let buf_2 = Buffer.allocUnsafe(10)
console.log(buf_2)

    使用这种方式创建 buffer 可能存在旧数据，可能会影响执行结果，所以叫 unsafe ，但是效率比 alloc 高.。

    因为内存空间是可以复用的，比如A程序要执行，等它执行完成之后，那么这段内存空间就没人用了，所以当B程序去执行的时候，就可以再次去使用这段内存空间。

    方式三：

    //TODO 方式三：
//通过字符串创建 Buffer
//<Buffer 68 65 6c 6c 6f>
let buf_3 = Buffer.from("hello")
console.log(buf_3)

//通过数组 创建Buffer
let buf_4 = Buffer.from([105, 108, 111, 118,101, 121, 111, 117])

    # Buffer 与字符串的转换

    可以借助 toString 方法将 Buffer 转为字符串

    toString 默认是按照 utf-8 编码方式进行转换的

    let buf_4 = Buffer.from([105, 108, 111, 118,101, 121, 111, 117])
console.log(buf_4)
//Buffer转字符串 iloveyou
console.log(buf_4.toString())

    # Buffer的读写

    Buffer 可以直接通过[] 的方式对数据进行处理。

    //通过字符串创建 Buffer
let buf_3 = Buffer.from("hello")

//TODO 读取

//TODO 修改

    注意:

    1. 如果修改的数值超过 255 ，则超过 8 位数据会被舍弃
    2. 一个 utf-8 的字符 一般 占3个字节

    # fs模块

    # 简介

    fs 全称为 file system ，称之为 文件系统 ，是 Node.js 中的 内置模块 ，可以对计算机中的磁盘进行操作。

    # 文件写入

    方法 说明
    writerFile 异步写入
    writerFileSync 同步写入
    appendFile / appendFileSync 追加写入
    createWriteStream 流式写入

    导入fs模块

    const fs = require('fs')

    # writerFile 异步写入

    语法： fs.writeFile(file, data[, options], callback)

    参数说明：

    • file 文件名
    • data 待写入的数据
    • options 选项设置 （可选）
    • callback 写入回调

    返回值： undefined

    // require 是 Node.js 环境中的全局变量， 用来引入模块
const fs = require("fs")

fs.writeFile("./霸道总裁爱上我.txt", "你的女朋友叫小芳", err => {
    //如果写入失败，调用回调函数，传入错误对象， 如果成功，传入 null
    if (err) {
        console.log(err)
        return
    }
    console.log("写入成功")
})

console.log(1)

    发现先打印的是 1，再显示 写入成功。 这就是异步写入， 不会等待其他线程的执行解构，直接执行后续的主线程代码，效率较好。

    # WriteFileSync同步写入

    语法： fs.writeFileSync(file, data[, options])

    参数与 fs.writeFile 大体一致，只是没有 callback 参数

    返回值： undefined

    // require 是 Node.js 环境中的全局变量， 用来引入模块
const fs = require("fs")

try {
    fs.writeFileSync("./霸道总裁爱上我Sync.txt", "你的女朋友叫小芳")
    console.log("写入成功")
} catch (err) {
    console.log(err)
}

console.log(1)

    发现先显示"写入成功"， 再打印“1”， 主线程 会等待 其他线程的执行结果，然后再继续执行主线程的代码。

    Node.js 中的磁盘操作是由其他 线程 完成的，结果的处理有两种模式：

    • 同步处理 JavaScript 主线程 会等待 其他线程的执行结果，然后再继续执行主线程的代码，效率较低
    • 异步处理 JavaScript 主线程 不会等待 其他线程的执行结果，直接执行后续的主线程代码，效率较好

    # 追加写入

    appendFile 作用是在文件尾部追加内容，appendFile语法与writeFile语法完全相同

    语法:

    • fs.appendFile(file, data[, options], callback)
    • fs.appendFileSync(file, data[, options])

    返回值： 二者都为 undefined

    // require 是 Node.js 环境中的全局变量， 用来引入模块
const fs = require("fs")


fs.appendFile("./霸道总裁爱上我.txt", ",好巧，我的女朋友也叫小芳", err => {
    if (err) throw err
    console.log("追加成功")
})

try {
    fs.appendFileSync("./霸道总裁爱上我Sync.txt", "\r\n他人： 我见她（同一个人）")
    console.log("appendFileSync追加成功")
}catch (e) {
    console.log(e)
}

    # 文件流写入

    语法： fs.createWriteStream(path[, options])

    参数说明：

    • path 文件路径
    • options 选项配置（ 可选 ）

    返回值： Object

    const fs = require("fs")

let ws = fs.createWriteStream("./感悟.txt")
ws.write("鹅鹅鹅,曲项向天歌\r\n")
ws.write("白毛浮绿水\r\n")
ws.write("红掌拨清波\r\n")
ws.end();

    程序打开一个文件是需要消耗资源的 ，流式写入可以减少打开关闭文件的次数。 流式写入方式适用于 大文件写入或者频繁写入 的场景, writeFile 适合于 写入频率较低的场景

    # Http

    # 简介

    HTTP（hypertext transport protocol）协议；中文叫超文本传输协议 是一种基于TCP/IP的应用层通信协议 这个协议详细规定了 浏览器 和万维网 服务器 之间互相通信的规则。

    协议中主要规定了两个方面的内容：

    • 客户端：用来向服务器发送数据，可以被称之为请求报文
    • 服务端：向客户端返回数据，可以被称之为响应报文

    报文：可以简单理解为就是一堆字符串

    # Http请求报文

    # 请求行

    # 请求方法

    请求方法 说明
    GET 主要用于获取数据
    POST 主要用于提交数据
    PUT 主要用于更新数据
    DELETE 主要用于删除数据
    HEAD 主要用于获取响应头
    OPTIONS 主要用于获取支持的请求方法
    CONNECT 主要用于建立连接隧道
    TRACE 主要用于追踪请求的传输路径

    # Get 和Post请求的区别

    get： 请求能够携带的参数比较少，大小有限制，会在浏览器的URL地址栏中显示数据内容，不安全，效率高

    Post:请求能够携带的参数没有限制，大小没有限制，不会在浏览器的URL地址栏中显示数据内容，安全，高效

    # 请求URL（统一资源定位器）

    https://www.baidu.com:80/index.html?a=100&b=100

    • http: 协议（https，ftp，ssh）
    • 域名：www.baidu.com
    • 端口：80
    • 路径： /index.html
    • 查询的字符串 a=100&b=100

    版本号 发布时间

    3 2018

    # 请求头

    格式：『头名：头值』 常见的请求头有：

    请求头 解释
    Host 主机名
    Connection 连接的设置 keep-alive（保持连接）；close（关闭连接）
    Cache-Control 缓存控制 max-age = 0 （没有缓存）
    Upgrade-Insecure-Requests 将网页中的http请求转化为https请求（很少用）老网站升级
    User-Agent 用户代理，客户端字符串标识，服务器可以通过这个标识来识别这个请求来自哪个客户端 ，一般在PC端和手机端的区分
    Accept 设置浏览器接收的数据类型
    Accept-Encoding 设置接收的压缩方式
    Accept-Language 设置接收的语言 q=0.7 为喜好系数，满分为1
    Cookie 后面单独讲

    # 请求体

    请求体内容的格式是非常灵活的， （可以是空）=> GET请求， （也可以是字符串，还可以是JSON）=> POST请求

    例如：

    • 字符串：keywords=你好&page=2
    • JSON：{"keywords":"手机","price":2000}

    # 响应行

    • 200：响应状态码
    • 404 Not Found
    • 500 Internal Server Error
    • 还有一些状态码，参考：https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Status

    1xx 信息响应

    响应状态码和响应字符串关系是一一对应的。

    # 获取HTTP请求报文

    想要获取请求的数据，需要通过 request 对象

    含义 语法 重点掌握
    请求方法 request.method *
    请求版本 request.httpVersion
    请求路径 request.url *
    URL 路径 require('url').parse(request.url).pathname *
    URL 查询字符串 require('url').parse(request.url, true).query *
    请求头 request.headers *
    请求体 request.on('data', function(chunk){})request.on('end', function(){});
    1. request.url 只能获取路径以及查询字符串，无法获取 URL 中的域名以及协议的内容
    2. request.headers 将请求信息转化成一个对象，并将属性名都转化成了『小写』
    3. 关于路径：如果访问网站的时候，只填写了 IP 地址或者是域名信息，此时请求的路径为『 / 』
    4. 关于 favicon.ico：这个请求是属于浏览器自动发送的请求

    # 获取请求头

const http = require('http');

    //获取请求方法
    console.log(request.method)
    //获取请求的url
    console.log(request.url)
    //获取HTTP协议的版本号
    console.log(request.httpVersion)

    # 获取请求体

    //1. 导入http 模块
const http = require('http');

//2. 创建服务对象，  create 创建 server 服务
const server = http.createServer((request, response) => {
    // 声明一个遍历
    let body = ''
    //绑定 data事件
    request.on('data', chunk => {
        //在对buffer进行加法运算时, 会自动调用 toString 方法
        body += chunk
    })

    //绑定end 事件
    request.on('end', () => {
        console.log(body)
        response.end("end")
    })
});

server.listen(80, ()=> {
    console.log("服务已经启动了")
})

    # 静态资源问题

    将html 改成 css外部引入发现问题

    # 问题分析

    http://www.inmind.com/ ==> 相当于在 03-获取Http响应报文这个文件夹下。

    请求 http://www.inmind.com/的时候 pathname= / 请求的是 login-静态资源问题.html 而html中 有<link rel="stylesheet" href="./css/login.css"> 这个语句。这个就会请求当前路径的/css/login.css 所以就会请求http://www.inmind.com/css/login.css

    相当于pathname = /css/login.css

    所以没有css 样式是因为，我们通过 外部嵌入的方式引入css， 没有将css 的 内容渲染到 响应体

    要解决这个问题， 不就相当于 当pathname = /css/login.css的时候 我们去 读取css文件，并且响应就好了

    # 解决

    //1. 导入http 模块
const http = require('http');
const fs = require("fs")

//2. 创建服务对象，  create 创建 server 服务
const server = http.createServer((request, response) => {

    let {pathname} = new URL(request.url, "http://www.inmind.com/")
    //http://www.inmind.com/
    if (pathname === "/"){
        //读取html文本内容的方式
        let html = fs.readFileSync(__dirname + "/login-静态资源问题.html")
        response.end(html)
    }
    //http://www.inmind.com/css/login.css
    else if(pathname === "/css/login.css") {
        //读取css文本内容的方式
        let css = fs.readFileSync(__dirname + "/css/login.css")
        response.end(css)
    }else {
        response.statusCode = 404;
        response.end('<h1>404 Not Found</h1>');
    }
});

server.listen(80, ()=> {
    console.log("服务已经启动了")
})

    3. 高维护性
